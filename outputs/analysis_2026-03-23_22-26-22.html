📝 What Was Done

The phone field was removed from the Address TypeScript interface in src/types.ts, and the corresponding phone input element was removed from src/components/CheckoutForm.tsx. This change was made for GDPR compliance as part of ticket KS-107.

Only 2 files were modified in the PR, but the phone field was used across at least 5 modules.

🔍 Root Cause / Risk Origin

The risk originates from the fact that the Address interface is a shared type imported by multiple modules across the project. Removing a field from a shared interface is a breaking change that propagates to every consumer of that interface.

The PR only updated 2 of the 5+ files that reference address.phone. This means 3+ files will have TypeScript compilation errors immediately after merge. Additionally, the backend API contract (POST /api/orders) may still expect a phone field, which would cause a 400 response or silent data loss.
